# Ereignisse
[⬅ Zurück zur Übersicht](../index.md)
## Einleitung
Dieses Kapitel beschreibt die Nutzung der Funktion **SRM > Ereignisse** zur Erfassung, Pflege und Nachverfolgung von Lieferantenereignissen.  
Ein Ereignis dient der Erinnerung an wichtige Vorgänge – z. B. Ablauf eines Zertifikats – und stellt sicher, dass alle relevanten Informationen zentral dokumentiert sind.  
Voraussetzung: Sie verfügen über die Berechtigung, Ereignisse im Modul **SRM** anzulegen und zu bearbeiten.

---

## Schritt-für-Schritt-Anleitung

### 1. Ereignis anlegen
1. Öffnen Sie **SRM > Ereignisse**.  
2. Bewegen Sie den **Mauszeiger** auf die **grüne Pinnadel** neben der **Ereignisübersicht**.  
3. Wählen Sie im **Vorschlagsfenster** **„Neues Ereignis“** und legen Sie im folgenden **Pop-up-Fenster** den **Ereignistyp** fest, z. B. **Zertifikat abgelaufen**.  
   - In **ADMIN > Properties > Ereignistypen** werden die Ereignisse definiert.  
     Der Typ mit der Sortierung 1 wird standardmäßig eingesetzt.  
     Wenn zwei Typen die gleiche Sortierung haben, wird alphabetisch priorisiert.  
4. Alternativ können Sie im **Vorschlagsfenster** direkt **Zertifikat abgelaufen** sowie den Zeitraum (z. B. 1 Tag, 7 Tage) auswählen.  
   Dadurch wird der **Ereignistyp** automatisch gesetzt und das **Datum** entsprechend angepasst.  
5. Pflegen Sie – falls erforderlich – **Firmenname**, **Verantwortlicher**, **Ansprechpartner** und **Kommentar**, damit das Ereignis vollständig dokumentiert ist.  
6. Klicken Sie anschließend auf **Speichern**.

**Hinweis:** Ereignisse können auch in den **Lieferantenstammdaten** angelegt werden. Dort befindet sich ebenfalls die **grüne Pinnadel** neben dem Lieferantennamen.
- Wird das Ereignis über **SRM > Ereignisse** angelegt, setzt das System den **Verantwortlichen** automatisch auf den aktuellen Benutzer.  
- Wird das Ereignis hingegen aus den **Lieferantenstammdaten** heraus erstellt, übernimmt das System die in den Stammdaten hinterlegte **Person des Vertriebsgebiets** als Verantwortlichen.

---

### 2. Ereignisübersicht und Bearbeitung
1. Öffnen Sie die **Ereignisübersicht** über **SRM → Ereignisse**.  
   Es werden alle Ereignisse angezeigt, für die Sie als **Verantwortlicher** eingetragen sind und deren Fälligkeitsdatum ≤ heute ist.  
2. Ergänzen Sie Texte über das **Bleistift-Symbol** in der Spalte **Kommentar**.  
3. Laden Sie Dokumente über die **Büroklammer** in der Spalte **Dokumente** hoch.  
   Verwenden Sie ggf. das **Upload-Symbol (grüner Pfeil)** → **„Klicken oder Dateien hierher ziehen“**, Datei auswählen → **Upload starten**.  
4. Ändern Sie bei erledigten Vorgängen den **Status** von *offen* auf *erledigt*.  
5. Wenn mehr Zeit für die Bearbeitung benötigt wird, nutzen Sie die Schnelleingaben **+1**, **+7**, **+14** oder **+28 Tage**.  
   Das **Fälligkeitsdatum** wird automatisch angepasst.  
6. Um aus einem Ereignis heraus einen Beleg zu erstellen (z. B. eine Bestellung), klicken Sie auf das **gelbe Beleg-Icon (Einkaufsbeleg anlegen)**.  
   Wählen Sie im Pop-up-Fenster den gewünschten **Belegtyp**; das System leitet Sie anschließend zur Seite **Einkaufsbeleg anlegen** weiter.

---

## Zusammenfassung
Die Ereignisfunktion unterstützt Sie bei der strukturierten Nachverfolgung von Lieferantenvorgängen.  
Pflegen Sie Verantwortliche, Termine und Kommentare sorgfältig, damit Ihr Team jederzeit den aktuellen Bearbeitungsstand erkennen kann.  
So bleibt Ihr Lieferantenmanagement transparent – von der Erinnerung bis zur abgeschlossenen Maßnahme.
